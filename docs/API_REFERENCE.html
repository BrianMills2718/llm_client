Module Summaries

llm_client is the shared public facade for provider transport, agent SDK routing, prompt rendering, observability, retries/fallbacks, and related runtime policy.

Source: llm_client/__init__.py

llm_client.client wraps litellm and agent SDKs.

It exposes the typed runtime entry points for sync, async, structured, batch, and streaming calls.

Source: llm_client/client.py

llm_client.io_log handles persistent I/O logging for LLM calls and embeddings.

It writes dated JSONL logs, optionally mirrors into SQLite, and keeps retention bounded.

Source: llm_client/io_log.py
